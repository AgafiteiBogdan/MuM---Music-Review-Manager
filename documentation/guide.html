Music Review Manager

Introducere

MuM sau Music Review Manager este o aplicatie care ajuta utilizatorii la gestionarea operelor muzicale in functie de mai multe criterii. Acesti utilizatori pot cauta piese noi dupa un anumit gen muzical sau de la un anume artist, pot cauta albume intregi si adauga piese noi in aplicatie. De asemnea, daca un utilizator crede ca poate imbunatatii aplicatia cu noi informatii, exista optiunea de a adauga comentarii la piese. Un alt sistem oferit de aplicatie e cel prin care se poate oferi note pieselor, si astfel va exista un rating de maxim 5 puncte.

Proiectarea Aplicatiei

Diagrame

Primul pas in care se iau decizii importante asupra aplicatiei este realizarea diagramelor, Usecase Diagram, care definesc interactiunea cu userul si fiecare scenariu posibil in aceasta aplicatie.

Mai jos sunt prezentate scenariile de utilizare a aplicatiei si o diagrama care ne ajuta sa intelegem mai bine interactiunea cu aceasta.

A. Un utilizator neinregistrat acceseaza aplicatia:

  • 1.Utilizatorul acceseaza aplicatia.
  • 2.Utilizatorul este directionat catre pagina principala.
  • 3.Utilizatorul vizualizeaza pagina cu optiunile de cautare a pieselor.
  • 4.Utilizator vizualizeaza lista cu ultimele piese introduse in aplicatie.
  • 5.Utilizatorul vizualizeaza pagina cu piesele filtrate dupa anumite criterii.

EXTENSII

  • 3a.Utilizatorul cauta informatii suplimentare despre o piesa.
  • 3a1.Utilizatorul este indrumat sa-si creeze un cont.
  • 3a1a.Utilizatorul nu are cont si va fi redirectionat catre pagina de Register.

B. Un utilizator inregistrat acceseaza aplicatia:

  • 1.Utilizatorul acceseaza aplicatia.
  • 2.Utilizatorul este directionat catre pagina principala.
  • 3.Utilizatorul acceseaza pagina de Login si introduce numele de utilizator si parola.
  • 4.Utilizatorul vizualizeaza pagina cu optiunile de cautare a pieselor..
  • 5.Utilizatorul cauta informatii suplimentare despre o piesa
  • 6. Utilizatorul evalueaza o piesa
  • 7.Utilizatorul adauga comentarii la o piesa

EXTENSII

  • 3a.Daca numele de utilizator sau parola sunt gresite, se va afisa un mesaj de eroare.
  • 7a.Utilizatorul este redirectionat catre platforma respectiva.

Urmatoarea diagrama este cea arhitecturala si ne prezinta tehnologiile folosite si cum interactioneaza ele.

Baza de date

Pentru stocarea informatiilor se foloseste o baza de date MySql. Primul pas in realizarea acestei Baze de Date a fost desenarea diagramei Entitate-Relatie care ne ofera absolut orice informatie despre tabelele ce trebuie create, coloane si chei. Mai jos este diagrama Entitate-Relatie a acestei baze de date.

Xampp - Apache

Fiecare aplicatie WEB are nevoie de un server pe care aceasta sa ruleze. Noi vom folosi aplicatia XAMPP si anume un server Apache care va rula PHP-ul din aceasta aplicatie. De asemenea, XAMPP ne ajuta si cu o baza de date MySql, care poate fi gestionata foarte usor din PHPMyAdmin.

PHP

Pentru a face conexiunea cu Baza de Date definita mai sus, vom folosi PHP. De asemenea PHP ne ajuta la inserarea tuturor informatiilor in aceasta baza de date si preluarea lor atunci cand este nevoie. PHP mai este folosit si pentru crearea unei sesiuni de Login, si setarea de Cookies. Astfel avem anumite selectii si preferinte ale utilizatorului mereu la indemana.

Javascript

Pentru partea de Frontend vom folosi Javascript, mai ales pentru validari si pop-up-uri dar si pentru anumite butoane care vor aparea doar la acei useri care sunt logati. De altfel, Javascript este folosit la majoritatea paginilor, acolo unde ultimele piese postate vor fi sortate dupa diferite criterii. Va fi implementata o pagina care la alegerea unor informatii despre o piesa, le sorteaza si le afiseaza doar pe cele dorite de user fara a fi nevoie sa dam refresh la toata pagina Web.

Ajax

Pentru pagina News se foloseste Ajax, astfel asigurand o filtrare rapida a pieselor dupa criteriile dorite fara a fi necesar sa incarcam din nou toata pagina. In acest mod si interactiunea dintre utilizator si aplicatie este una mai prietenoasa.
